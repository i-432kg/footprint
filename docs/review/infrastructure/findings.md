| 1 | 依存と責務分離 / 一貫性 | `src/main/java/jp/i432kg/footprint/infrastructure/datasource/mapper/repository/PostMapper.java`<br>`src/main/java/jp/i432kg/footprint/infrastructure/datasource/mapper/repository/ReplyMapper.java`<br>`src/main/java/jp/i432kg/footprint/infrastructure/datasource/mapper/repository/UserMapper.java` | Medium | repository mapper の Insert 用 DTO が、application/domain 側で決めた時刻を使わず `LocalDateTime.now()` を再生成している。`PostMapper` は `createdAt` / `updatedAt` / `post_images.createdAt` を、`ReplyMapper` は `createdAt` / `updatedAt` を永続化時刻で上書きしており、ユースケース側で組み立てた値と DB 保存値がずれる。さらに `UserMapper` は新規作成直後の `lastLoginAt` まで `now()` で埋めており、未ログインユーザーを「ログイン済み」と同じ状態で保存してしまう。 | 時刻の決定責務をどの層に置くかを統一する。application/domain で生成した値を採用するなら mapper はそれを透過的に保存し、DB 既定値で扱うなら上流モデルから timestamp を外す。少なくとも `lastLoginAt` は初回ログイン前状態を正しく表現する値へ改める。 | クローズ | `PostMapper` / `ReplyMapper` は上流で生成した `createdAt` を保存する形に修正済み。`UserMapper` は新規作成時に `lastLoginAt` を保存せず、認証成功時に `LastLoginRecorder` から `AuthMapper.updateLastLoginAt(...)` で更新する構成へ整理済み。 |
| 2 | 不変条件・境界値 / 一貫性 | `src/main/java/jp/i432kg/footprint/infrastructure/seed/local/LocalSeedService.java`<br>`src/main/java/jp/i432kg/footprint/infrastructure/seed/stg/StgSeedService.java` | Medium | seed サービスは `CreateReplyCommand.of(...)` に `null` を渡してルート返信を作っているが、現行設計では `ParentReply.root()` を使って `null` を状態表現にしない方針になっている。presentation 経路はすでに `ParentReply` に寄せているため、seed 経路だけが application の non-null 契約を破っており、境界方針が崩れている。 | seed サービスも `ParentReply.root()` を使って `CreateReplyCommand` を組み立てる。`null` を使う旧経路が他に残っていないかも併せて確認する。 | クローズ | fixed seed scenario を共通化した `AbstractFixedSeedService` で `ParentReply.root()` / `ParentReply.of(...)` を使う実装へ統一済み。`CreateReplyCommand` へ `null` を渡す旧経路も再確認し、残存なし。 |
