## 追加で確認したい観点
- トランザクション整合性: 画像保存成功後に DB 永続化が失敗した場合の補償処理や orphan データの扱い。
- 例外公開ポリシー: `ProblemDetail.details` に何を載せてよいかの基準、PII / 秘密情報のマスキング方針。
- 永続化往復整合: type handler / mapper を経由した値オブジェクトの round-trip で正規化や値欠損が起きないか。
- テスト容易性: `LocalDateTime.now()` や `UlidCreator` 直呼びを `Clock` / `IdGenerator` へ抽象化すべきか。
