# UT実装指針

## 基本方針

- `docs/ut` 配下に、対象パッケージと同じ構成のディレクトリを作成する。
- `ファイル名_test_design.md` という命名で、テスト仕様書を作成する。
- テストケースは、正常系・異常系・エッジケースに分類して記載する。
- テスト仕様書に基づいた UT を `test` パッケージ配下に作成する。
- テストを実行し、カバレッジを確認する。
- C1 カバレッジを採用し、可能な限り 100% を目指す（努力目標）。
- 同値分割・境界値分析を用いたブラックボックステストとする。

## コーディングルール

- テスト名は `should_期待する動作` とする。
- JavaDoc でテスト内容の概要を 1 行程度で簡潔に記述する。
