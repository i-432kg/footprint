# Security Policy

## 概要

本リポジトリは個人開発中のプロジェクトです。  
セキュリティ上の問題と思われる点を見つけた場合は、公開の場ではなく非公開でご連絡いただけると助かります。

本リポジトリは現在開発・検証段階のため、セキュリティ修正はベストエフォートで対応します。

## サポート対象バージョン

現時点では、最新の開発ブランチを中心に確認・対応を行います。

| バージョン / ブランチ | サポート状況 |
| --- | --- |
| develop | ベストエフォートで対応 |
| それ以前のブランチ・過去コミット | 対応対象外 |

## 脆弱性の報告方法

脆弱性の可能性がある問題を発見した場合は、**公開 Issue には投稿しないでください**。  
内容を確認のうえ対応したいため、以下のいずれかの方法で非公開にてご連絡ください。

- GitHub の Private Vulnerability Reporting
- 連絡先メールアドレス: `432kg.biz@gmail.com`

報告時には、可能な範囲で以下の情報を含めてください。

- 対象リポジトリ名
- 脆弱性の概要
- 再現手順
- 想定される影響範囲
- 参考情報（ログ、スクリーンショット、修正案など）

## 対応方針

報告を受けた後、内容を確認し、必要に応じて再現確認・影響調査を行います。  
有効な問題であると判断した場合は、修正または緩和策の検討を行います。

目安として、以下の方針で対応します。

- 受領確認: 3営業日以内を目安
- 内容確認・一次回答: 状況に応じて順次対応
- 修正対応: 影響度と再現性を踏まえて判断

ただし、個人開発プロジェクトのため、必ずしも迅速な対応を保証するものではありません。

## 公開に関する方針

利用者保護のため、**修正または緩和策が用意される前の公開開示はお控えください**。  
問題への対応後、必要に応じて概要と修正内容を公開する場合があります。

## 対象範囲

本ポリシーは、このリポジトリのソースコードおよび設定ファイルを対象とします。  
以下は原則として対象外です。

- 開発中・検証中の一時的なコード
- ローカル環境固有の設定
- 外部サービスや外部ライブラリ自体の脆弱性
- 想定外の利用方法のみで成立する軽微な問題

## 免責事項

本リポジトリはポートフォリオ・学習・検証用途を含む個人開発プロジェクトです。  
そのため、商用サービスと同等のセキュリティ対応体制や SLA は提供していません。
